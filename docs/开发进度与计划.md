#### 3. 飞书全线对接 ✅

| 模块 | 实现 |
|------|------|
| **@larksuite/cli** | 全局安装 v1.0.54，assistant profile 已配置（App ID: cli_a93bc6d586785cb2），bot 身份就绪 |
| **@larksuiteoapi/node-sdk** | v1.67.0，WSClient + EventDispatcher 实现 WebSocket 长连接 |
| **Gateway URL Challenge** | `POST /feishu/event` 检测 `type: "url_verification"` → 返回 `{ challenge }` |
| **Gateway 签名校验** | `X-Lark-Signature` + HMAC-SHA256 验证 |
| **事件回路闭合** | `/feishu/event` 响应后异步执行 Agent Loop → lark-cli 回复 |
| **chat_id 自动记忆** | `chatSessions` Map 在每次收到事件时自动登记，注入 System Prompt |
| **WS 事件格式转换** | 长连接扁平结构 `{ event_id, message, sender }` → HTTP 回调嵌套格式 `{ header: {...}, event: {...} }` |
| **WebSocket 长连接** | WSClient 主动连接飞书，实时接收事件，SDK 内置心跳/重连，失败自动降级轮询 |
| **轮询引擎（降级）** | 每 5 秒拉取所有会话消息，冗余去重，跳过 bot 自身消息 |

#### 4. 通信架构（内网部署方案）

```
方案 1（主）：WebSocket 长连接
  飞书服务器 ←──wss── jarvis-team (内网)
  实时推送，<1秒延迟，SDK 内置心跳+重连
  失败自动降级 ↓

方案 2（降级）：主动轮询
  jarvis-team ──每5秒──→ lark-cli im +chat-messages-list
  拉取所有会话消息，去重后处理
  不需要公网 URL，不需要事件回调
```
