## 架构总览

飞书只作为前台入口，不承载复杂业务编排。所有 Agent 协作通过 `Orchestrator + Agent Loop` 中心化收敛。

v1 核心链路：

```
Feishu Bot (每个 Agent 独立 Bot)
       │ HTTP Webhook /feishu/bot/{agentId}
       ▼
┌──────────────┐
│   Gateway    │  去重 + 签名校验 + 事件标准化
└──────┬───────┘
       │ InternalEvent
       ▼
┌──────────────┐
│ Orchestrator │  意图分类 + 步骤规划
└──────┬───────┘
       │ Task + Steps
       ▼
┌────────────────────────────────────────┐
│           Agent Loop                   │
│  while(true) {                         │
│    think → act → observe               │
│  }                                     │
│  ├─ 保险丝 1: 死循环检测 (SHA256 指纹)   │
│  ├─ 保险丝 2: Token 预算 (90% nudge)    │
│  └─ 保险丝 3: 截断恢复 (三步递进)        │
│  7 种退出路径                           │
└──────┬─────────────────────────────────┘
       │ 工具调用
       ▼
┌────────────────────────────────────────┐
│         Tool System (7 步管线)          │
│                                         │
│  Zod 验证 → 业务校验 → 输入补全 →        │
│  Hook → 权限检查 → 执行+截断 → Hook     │
└──────┬─────────────────────────────────┘
       │ 飞书 Open API（用 Bot 自己的凭证）
       ▼
  飞书群消息 / 文档 / 卡片
```

## 当前 v1 范围

### Agent 角色（3 个）

| Agent | 可见性 | 职责 | 可主动发言 |
|-------|--------|------|-----------|
| Secretary | 前台 | 接需求、起草、总结、汇报 | ✅ |
| Reviewer | 前台 | 质量复核、风险发现 | ❌（仅发现问题时） |
| Orchestrator | 后台 | 意图识别、任务路由 | ❌ |

### 存储层

| 数据 | 当前方案 | 后续方案 |
|------|---------|---------|
| Agent 配置 | JSON 文件 `agents/*.json` | PostgreSQL |
| Session/Task | 内存 Map | PostgreSQL |
| 记忆 | 未实现 | 文件/DB |

## 协作模式

| 模式 | 协调方式 | 推荐用途 | 状态 |
|------|----------|---------|------|
| `single-agent` | 单上下文串行 | 轻任务、短路径执行 | ✅ v1 |
| `parent-child` | Orchestrator 派子 Agent，摘要回传 | 调研、长任务分段 | Phase 2 |
| `light-swarm` | 共享任务表 + mailbox | Phase 2 | Phase 3 |
