# 飞书多Agent协作系统 技术架构

> 本文档从 Obsidian 项目文档同步。源文件：[飞书多Agent协作系统-技术架构](飞书多Agent协作系统-技术架构.md)

## 架构总览

飞书只作为前台入口，不承载复杂业务编排。所有 Agent 协作通过 `Orchestrator + State Store` 中心化收敛。

```
Feishu Group → Gateway → Orchestrator → Agent Runtime → Tool Service → Feishu API
                                │
                          State Service (Session/Task/Memory/Mailbox)
```

## 协作模式

| 模式 | 协调方式 | 推荐用途 |
|------|----------|---------|
| `single-agent` | 单上下文串行 | 轻任务、短路径执行 |
| `parent-child` | Orchestrator 派子 Agent，摘要回传 | 调研、长任务分段 |
| `light-swarm` | 共享任务表 + mailbox | Phase 2 |

## 实施顺序

1. gateway + orchestrator + state + secretary
2. pm-agent + qa-risk-agent
3. tool policy + prompt pipe + memory + 保险丝
4. Docs + 多维表格 + 卡片审批
